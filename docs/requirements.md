# 要件定義書：チーム日報管理システム

**プロジェクト名:** devstep-daily-report  
**作成日:** 2026-03-16  
**実装期間:** Week3〜Week8（6週間）

---

## 1. サービス概要

企業・チームで使用する日報管理Webアプリケーション。  
メンバーが日々の業務内容を記録し、チーム内で共有・コメントできる。

---

## 2. ターゲットユーザー

- 中小企業のチームリーダー・マネージャー
- リモートワークチームのメンバー

---

## 3. 解決する課題

| 課題 | 解決策 |
|---|---|
| 日報を紙・Excelで管理している | Webで一元管理 |
| チームメンバーの業務内容が見えない | 全員の日報を一覧表示・共有 |
| 日報へのフィードバックが遅い | コメント機能で迅速化 |

---

## 5. 必須機能一覧

### 5.1 ユーザー認証

- メール/パスワードによる新規登録・ログイン・ログアウト
- パスワードリセット（メール送信）
- 未ログイン時は `/login` へリダイレクト
- セッション切れ時の自動リダイレクト

### 5.2 プロフィール管理

- 表示名の編集
- アバター画像のアップロード・変更
- メールアドレスの表示（読み取り専用）

### 5.3 日報 CRUD

**作成（Create）**
- タイトル（必須・50文字以内）
- 日付（必須・デフォルトは今日）
- カテゴリ（選択式：開発 / 会議 / その他）
- 内容（必須・2000文字以内）
- バリデーションエラーは画面に表示

**一覧表示（Read）**
- 全チームメンバーの日報を表示
- 表示項目：タイトル・投稿者名・日付・カテゴリ・本文冒頭（100文字）・コメント数
- デフォルトは作成日時の降順（新しい順）
- ページネーション（1ページ 10件）

**詳細表示**
- タイトル・投稿者・日付・カテゴリ・本文全文・作成日時・更新日時
- コメント一覧
- 編集・削除ボタン（自分の日報のみ表示）

**編集（Update）**
- 自分が作成した日報のみ編集可能

**削除（Delete）**
- 自分が作成した日報のみ削除可能
- 削除確認ダイアログを表示

### 5.4 検索・フィルター

- 日付範囲検索（開始日・終了日）
- ユーザー絞り込み
- カテゴリ絞り込み
- キーワード検索（タイトル・本文の部分一致）

### 5.5 コメント機能

- 日報詳細画面からコメント投稿（500文字以内）
- コメント一覧表示（投稿者名・アバター・本文・投稿日時）
- コメント削除（自分が投稿したコメントのみ）

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| 認証 | Supabase Auth（メール/パスワード） |
| 認可 | Supabase RLS によるレコードレベル制御 |
| バリデーション | フロントエンド（リアルタイム）+ バックエンド（RLS・DB制約）|
| エラーハンドリング | 全API呼び出しで例外処理・ユーザーへのエラー表示 |
| レスポンシブ | スマホ（375px〜）・タブレット（768px〜）・デスクトップ（1024px〜）|
| TypeScript | `strict: true`・`any` 型禁止 |
| テスト | ユニット（Vitest）+ E2E 最低1シナリオ（Playwright）|
| CI/CD | PR時：ビルド・Lint・テスト自動実行 / main マージ時：Vercel 自動デプロイ |
| セキュリティ | 未ログイン時リダイレクト・セッション切れ対応・RLS による権限制御 |

## 8. データベース設計（概要）

| テーブル | 主なカラム |
|---|---|
| users（Supabase Auth管理） | id, email, name, avatar_url, created_at |
| daily_reports | id, user_id, title, date, category, content, created_at, updated_at |
| comments | id, report_id, user_id, content, created_at |

### RLS ポリシー

- `daily_reports`：全員が全件閲覧可能 / 自分のレコードのみ作成・編集・削除可能
- `comments`：全員が全件閲覧可能 / 自分のレコードのみ作成・削除可能
